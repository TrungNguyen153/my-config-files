# Share nvim config between WSL and Windows host — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `~/.config/nvim` in WSL Ubuntu point at the same real folder as the existing Windows Junction (`%LOCALAPPDATA%\nvim` → `D:\Workspace\my-config-files\.config\nvim`), so a single config tree drives both `nvim.exe` (Windows fallback) and `nvim` inside WSL (daily driver, attached via `neovide --wsl`). Resolve the lazy-lock conflict by routing each OS to its own committed lockfile.

**Architecture:** Sibling symlinks (not chained) onto a single real folder under `D:\`. Per-OS lockfile dispatch in `init.lua` via `lazy.nvim`'s `lockfile` option. Shell branched per OS (`nu` on Windows, `bash` on WSL). Plugin data, Mason binaries, and treesitter parsers naturally diverge because `stdpath('data')` differs per OS — no code needed there.

**Tech Stack:** Neovim ≥ 0.11, lazy.nvim (already in use), Neovide on Windows, WSL Ubuntu, Nushell on Windows / bash on WSL, scoop-installed `nvim.exe` and `neovide.exe`.

**Spec:** `docs/superpowers/specs/2026-05-12-nvim-wsl-config-share-design.md`

## File Map

| Path (under `D:\Workspace\my-config-files\.config\nvim\`) | Action | Responsibility |
|---|---|---|
| `init.lua` | Modify (~5 lines) | Detect WSL; dispatch lockfile name to `lazy.setup` |
| `lua/settings/general.lua` | Modify (~10 lines) | Branch `vim.opt.shell` per OS |
| `lazy-lock.json` | Rename → `lazy-lock.windows.json` | Existing Windows pins become explicitly OS-tagged |
| `lazy-lock.wsl.json` | Generated by first WSL `:Lazy sync` | WSL pins, committed |
| `.gitattributes` | Create (repo root, not under `.config/nvim`) | Pin LF line endings for Lua/JSON; avoid CRLF surprises in shared tree |

Filesystem (outside the repo):
- WSL: create `~/.config/nvim` → `/mnt/d/Workspace/my-config-files/.config/nvim` (sibling symlink, not a chain)

## Conventions used in this plan

- Commands prefixed `PS>` run in **PowerShell on Windows** (working directory: `D:\Workspace\my-config-files`).
- Commands prefixed `WSL$` run in **WSL Ubuntu** (start with `wsl.exe` from PowerShell, or open a WSL terminal).
- All paths inside the config tree are relative to `D:\Workspace\my-config-files\.config\nvim\` unless otherwise stated.
- The repo root for git operations is `D:\Workspace\my-config-files\`.

## Task 1: Add `.gitattributes` to pin line endings

**Why first:** the committed spec already triggered a `LF will be replaced by CRLF` warning. We're about to add JSON lockfiles edited by Linux tools (`lazy.nvim` on WSL) and Lua files edited from either side. CRLF roundtripping causes "modified" status noise and occasional plugin-parser bugs. Pin once.

**Files:**
- Create: `D:\Workspace\my-config-files\.gitattributes`

- [ ] **Step 1: Create `.gitattributes` at the repo root**

Path: `D:\Workspace\my-config-files\.gitattributes` (NOTE: repo root, not under `.config\nvim`).

Content:

```
* text=auto eol=lf
*.lua text eol=lf
*.json text eol=lf
*.toml text eol=lf
*.md text eol=lf
*.sh text eol=lf
*.scm text eol=lf
```

- [ ] **Step 2: Renormalize existing tracked files**

```
PS> cd D:\Workspace\my-config-files
PS> git add --renormalize .
PS> git status
```

Expected: list of files staged as modified (line-ending normalization). If your local `core.autocrlf` is `true`, working-copy files keep CRLF, but the index now stores LF. That's the goal.

- [ ] **Step 3: Commit**

```
PS> git add .gitattributes
PS> git commit -m "chore: pin LF line endings via .gitattributes"
```

Expected commit summary: `.gitattributes` plus any renormalized files. Verify:

## Task 4: Branch the shell setting per OS

**Files:**
- Modify: `D:\Workspace\my-config-files\.config\nvim\lua\settings\general.lua` (lines 5–22, the `powershell_options` block)

Reference (current state):

```lua
-- default shell
local powershell_options = {
    -- shell = vim.fn.executable "pwsh" == 1 and "pwsh" or "powershell",
    -- ... commented options ...

    -- This for NuShell
    shell = 'nu',
    shellcmdflag = '-c',
    shellquote = '',
    shellxquote = '',
    shellslash = true,
}
for option, value in pairs(powershell_options) do
    vim.opt[option] = value
end
```

- [ ] **Step 1: Replace the block with per-OS branching**

Replace lines 5–22 of `lua/settings/general.lua` with:

```lua
-- default shell, branched per OS
-- Windows host -> Nushell. WSL/Linux -> bash. nu is not on PATH inside WSL Ubuntu
-- by default; using it there breaks every shell-out plugin (toggleterm, conform,
-- rustaceanvim executor, mason installers, gitsigns, snacks find-in-files, etc.).
local is_wsl = vim.fn.has('wsl') == 1
local is_win = vim.fn.has('win32') == 1

if is_win and not is_wsl then
    vim.opt.shell = 'nu'
    vim.opt.shellcmdflag = '-c'
    vim.opt.shellquote = ''
    vim.opt.shellxquote = ''
    vim.opt.shellslash = true
else
    vim.opt.shell = 'bash'
    vim.opt.shellcmdflag = '-c'
end
```

- [ ] **Step 2: Smoke-test on Windows**

```
PS> nvim --headless +'lua print("shell=", vim.o.shell)' +qa
```

Expected stdout: `shell=\tnu`. Confirms Windows path branch fired.

- [ ] **Step 3: Open a Windows nvim and verify `:terminal` still opens Nushell**

```
PS> nvim
:terminal
```

You should see a Nushell prompt inside the terminal buffer. Type `version` and Enter — should print Nushell version. `<C-\><C-n>` then `:bd!` to close, `:qa` to exit.

- [ ] **Step 4: Commit**

